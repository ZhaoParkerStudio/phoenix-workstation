# 电视新闻文稿生成 Skill（完整版）

## 一、任务定义

根据用户提供的新闻素材（新华社通稿、外电报道、视频脚本、社交媒体信息等），按照**新华社风格**和**电视新闻播出标准**，生成完整的电视新闻文稿，包括：

- **口播**：主播出镜导语
- **LVO**：配音正文
- **正文**：剪辑脚本（含时间码和画面说明）
- **字幕**：主标题（Content）和副标题（Heading）
- **全文文稿**：逐句分行版

## 二、触发方式

用户可通过以下指令触发本 Skill：

- "写LVO"
- "按照新华社风格写作一条LVO"
- "根据素材写电视新闻稿"
- 直接提供新闻素材并说"生成稿件"
- "帮我写一条关于XXX的新闻稿"

## 三、输入素材处理规则

### 3.1 素材来源优先级

1. **新华社通稿**（最优先采用，严格遵循口径）
2. **央视/官方媒体**（第二优先级）
3. **外电报道**（路透、美联、法新、韩联社、共同社等）
4. **视频SHOTLIST描述**
5. **社交媒体信息**

### 3.2 信息核实原则

- 所有事实必须有据可查
- 具体数据（时间、人数、金额等）需标注来源
- 不能添加任何推测性内容
- 如素材中存在矛盾信息，以新华社口径为准

### 4.2 【口播】写作规则

- **单句或两句话**，用逗号自然断句
- **不超过60字**
- **包含要素**：时间（具体日期或"近日"）、人物（职务+姓名）、核心动作、关键结果
- **不使用直接引语**
- **示例**：
  > "新華社洛杉磯6月20日電，洛杉磯市一處大型商業倉儲設施火災持續數日，市長巴斯20日宣佈該市進入緊急狀態。"

### 4.3 【LVO】写作规则（核心）

#### 时长与字数控制

| 秒数 | 建议字数范围 |
|------|-------------|
| 30秒 | 140–160字 |
| 40秒 | 190–210字 |
| 45秒 | 220–240字 |
| 50秒 | 250–270字 |
| 55秒 | 280–300字 |
| 60秒 | 310–330字 |

#### 结构与叙事顺序

**优先顺序**：
1. **画面描述先行**（如有视频素材）：先描述观众看到的画面
2. **核心事件陈述**：时间、地点、人物、事件
3. **细节展开**：数据、引语、背景
4. **相关方回应**：官方表态、专家分析
5. **背景补充**：历史数据、对比信息

#### 语言规范

| 维度 | 要求 | 正确示例 | 错误示例 |
|------|------|----------|----------|
| 时态 | 过去时/现在完成时 | "表示""說""已" | "將要""會" |
| 时间 | 具体到月日 | "6月20日" | "昨天""今天" |
| 人名 | 首次全名+职务，后续简称 | "美國國防部長赫格塞斯"後"赫格塞斯" | 全程用全名 |
| 数字 | 大数字用"萬""億" | "1億零654萬" | "1.0654亿" |
| 百分比 | 用"百分之" | "百分之75" | "75%" |
| 引语 | 间接引语为主，关键句用引号 | "稱'朋友之間應當坦誠'" | 大量堆砌引语 |
| 连接 | 自然衔接 | "此外""同時""其中""目前" | "而且""但是" |

#### 长定语处理

长定语用逗号或顿号分隔，便于配音员换气：
> "來自韋拉克魯斯州山區的、21歲運動員塞萬提斯，因奪得控球技巧比賽冠軍，獲得6月11日在墨西哥城阿茲台克體育場舉辦的、世界杯揭幕戰首張門票。"

### 4.4 【正文】剪辑脚本规则

- **格式固定**：`【正文】编号 請剪X+Y=Z秒`
- **分段标注**：每段标注时长（秒）和画面说明
- **素材标识**：`<Clip>來源_事件_序號 (時間碼)</Clip>`
- **同一素材多个时间片段**用 `+` 连接
- **示例**：
  ```
  15秒
  <Clip>REUTERS_MUNICH_TRAIN_CRASH_20260620_001 (00:00:00-00:00:15)</Clip>
  +
  <Clip>REUTERS_MUNICH_TRAIN_CRASH_20260620_002 (00:00:15-00:00:30)</Clip>
  ```

### 4.5 【字幕】规则

**重要**：格式约定与传统相反！

| 字段 | 标签 | 作用 | 字数限制 |
|------|------|------|----------|
| **主标题** | `Content` | 概括核心事件（人物+动作） | 10字以内 |
| **副标题** | `Heading` | 补充具体细节（数据、引语、背景） | 15字以内 |

#### 生成逻辑

**Content（主标题）规则**：
- 主语 + 谓语 + 核心宾语
- 不加标点
- 优先动词：訪問、會見、抵達、宣佈、發生、致（數字）死（數字）傷

**Heading（副标题）规则**：
- 优先选择唯一性信息（"首個""首次"）
- 优先选择冲突性信息
- 优先选择直接引语中的关键句
- 可包含数据、时间、人物身份

**示例**：

| Content（主标题） | Heading（副标题） |
|-------------------|-------------------|
| 王毅會見巴基斯坦副總理兼外長達爾 | 推進中巴經濟走廊建設 巴方稱中方倡議正當其時 |
| 墨西哥總統贈出美加墨世界杯首張門票 | 21歲土著女運動員獲揭幕戰00001號門票 |
| 洛杉磯倉庫大火持續數日 | 建築儲存3.86萬噸冷凍食品 牆體泡沫緩慢燃燒 |

### 4.6 【全文文稿】规则

- 将口播和LVO按**自然分句**拆成短行
- 每行末尾加 `櫔 (())`
- 口播结束后空一行，再写LVO
- 以 `((END))` 单独一行结尾

## 五、新华社风格规范

| 维度 | 要求 |
|------|------|
| 客观性 | 避免主观评价，只陈述事实 |
| 政治术语 | 固定表述不可改（"中美元首會談"不用"習特會"） |
| 涉台用语 | "台灣地區""民進黨當局" |
| 国家全称 | 首次全称，后简称（"朝鮮民主主義人民共和國"後"朝方"） |
| 标点 | 中文标点，句号、逗号、顿号规范 |
| 引语 | 雙引號用「」或"" |
| 时间顺序 | 按事件发生顺序排列 |

## 六、LVO开头画面描述规则

如素材包含视频画面（SHOTLIST），LVO开头应先用1-2句话描述画面，再转入事件叙述：

**示例**：
> "慕尼黑一處鐵路橋下，兩節貨運車廂墜落街頭，車體因撞擊嚴重扭曲變形，碎片散落一地。警方已封鎖周邊道路，應急車輛在現場待命。當地時間20日凌晨1時40分許，兩列貨運火車疑似相撞後脫軌……"

## 七、特殊情况处理

### 7.1 新华社通稿直接改写
- 保留所有关键数据、地名、人名
- 将书面语长句拆成口语短句
- 删除不必要的修饰词

### 7.2 外电报道处理
- 注明消息来源（"據路透社報道"）
- 核实与中国立场相关的表述
- 对外国官员原话用"稱""說"转述

### 7.3 数据密集型新闻
- 只保留最核心数据
- 避免堆砌数字
- 可用"約""近"等概数词

### 7.4 突发灾害/事故
- 第一句交代核心（何事、何地、何果）
- 第二句展开细节（伤亡、救援）
- 第三句补充背景
- 避免渲染性语言

## 八、执行检查清单

生成稿件后逐项确认：

- [ ] 口播+LVO总时长符合用户要求
- [ ] 无错别字、漏字、标点正确
- [ ] 数字格式规范（万、亿、百分之）
- [ ] 人名、头衔、地名完整准确
- [ ] 时间表述具体（6月20日）
- [ ] 无主观评价词汇
- [ ] 关键引语有雙引號并标明说话人
- [ ] 背景信息与核心事实逻辑连贯
- [ ] 每句不超过35字
- [ ] 全文文稿每行正确添加 `櫔 (())`
- [ ] Content ≤10字，Heading ≤15字
