# SKILL：SOT新闻稿写作技能包

## 一、技能概述

本技能用于撰写**凤凰卫视风格的SOT电视新闻配音稿**，适用于国际时政、军事外交、突发事件等各类新闻场景。稿件采用**新华社风格**——客观、准确、简练，结构固定，逻辑清晰。

## 二、稿件整体结构（模板）

```
【口播】
（核心事实概括，80-120字，以"-VT-"结尾）

【正文】7-7164
（配音正文，300-600字，分段叙述，每段后标注<Clip>素材）
……
鳳凰衛視 綜合報道

【同期聲】
（可选，汇总关键引语）

【字幕】
<CG=橫題>
ID_Number:1
Heading:（主标题/肩题，10-20字）
Content:（副标题/主标题，10-25字）
Reporter:（记者名）

【全文文稿】
櫔
((VT))
((LVO))
((SB))
((UPSOUND))
((Standup))
((END))
```

## 三、各部分写作规范

### 1. 口播
- **功能**：一句话或两句话概括核心新闻点，抓住观众注意力。
- **格式**：纯文字，末尾加"-VT-"或"-LVO-"。
  - `-VT-`：纯配音，无画面配合。
  - `-LVO-`：配音配合画面（LVO为"Live Voice Over"的缩写）。
- **长度**：80-120字。
- **内容要素**：谁 + 什么时候 + 做了什么 + 有什么影响。

### 2. 正文（配音稿）
- **标识**：`【正文】7-7164`（固定格式）。
- **语言**：新华社风格，客观中立，避免主观评价。
- **长度**：300-600字。
- **结构**：
  - 第一段：展开核心事实。
  - 中间段落：按逻辑顺序（时间、重要性）补充背景、各方表态、影响。
  - 最后一段：总结或过渡到下一新闻点（如需要）。
- **转题**：如果新闻有多条不同主题，中间可加`（轉題加效）`或`（轉題不加效）`分隔。

### 3. 用片（Clip）标注
每段配音结束后，紧跟该段对应的视频素材标注。

**格式：**
```
<Clip>文件名_时间码</Clip> (画面描述)
```
- **文件名**：素材原始文件名（保留）。
- **时间码**：可选，如`(00:12:08-00:43:00)`。
- **画面描述**：简短说明画面内容，用括号括起来。
- **高清资料片**：标注为`(常用高清資料片)xxx`。
- **同一段内多个clip**：每行一个，依次列出。

### 4. 同期声（SB）
- **用途**：插入人物原声讲话（Soundbite）。
- **格式**：
  - 正文中标注：`SB 人物名 职务` 或 `SB1 人物名`
  - 然后列出`<Clip>`标注原声素材。
  - 随后提供中文翻译，放在括号内。
- **位置**：
  - 可嵌入正文中（紧跟人物出现的位置）。
  - 或集中在正文末尾的`【同期聲】`部分。
- **字幕标注**：同期声下方可加`<CG=左二行>`标注讲话人姓名和职务。

### 5. 字幕（CG）
- **格式**：使用`<CG=橫題>`标签。
- **Heading**：主标题/肩题，10-20字，概括核心事件。
- **Content**：副标题/主标题，10-25字，补充关键信息。
- **多题**：如有多个新闻点，可用`ID_Number:1`、`ID_Number:2`分别标注。
- **其他CG**：
  - `<CG=右一行>`：标注"資料片段"。
  - `<CG=左二行>`/`<CG=右二行>`：标注讲话人姓名和职务。

### 6. 全文文稿（底部标记）
- **开头**：`櫔`（固定符号）。
- **内部标记**：
  - `((VT))`：纯配音段落。
  - `((LVO))`：配音+画面段落。
  - `((SB))`：同期声段落。
  - `((UPSOUND))`：环境音段落。
  - `((Standup))`：记者出镜段落。
  - `((END))`：稿件结束。

## 四、写作风格（新华社风格）

| 要求 | 说明 |
|------|------|
| **客观中立** | 避免"胜利""失败""成功"等倾向性词汇；争议事件只陈述各方表态，不自行判断。 |
| **事实准确** | 严格依据参考素材，不添加未确认信息；数据、时间、地点、人名必须核对。 |
| **语言简练** | 多用短句，避免长定语；每段一般2-3句话。 |
| **逻辑清晰** | 按"总—分"或"时间顺序"组织；转题处用转题标识。 |
| **数字规范** | 年份用"2026年"，数量用"12人""5000名""1.5億"，百分比用"5.7%"。 |
| **译名统一** | 外国人名使用通用规范译名（如"特朗普"而非"川普"）。 |

## 五、工作流程

### Step 1：研读素材
- 阅读所有提供的参考素材（新华社通稿、外电报道、视频描述、shotlist等）。
- 提取核心事实、关键数据、各方表态、时间地点人物。

### Step 2：确定结构
- 决定是单条SOT还是多条合并。
- 如有多条，确定每条的主题和转题方式。
- 确定口播核心内容。

### Step 3：写口播
- 用1-2句话概括核心新闻点。
- 末尾加`-VT-`或`-LVO-`。

### Step 4：写正文
- 第一段展开核心事实。
- 中间段落补充背景、细节、各方表态。
- 每段后紧跟`<Clip>`素材标注。
- 保留关键同期声内容。

### Step 5：标注素材
- 对照shotlist和提供的clip文件名，标注每段对应的素材。
- 高清资料片注明`(常用高清資料片)`。
- 如需转题，在适当位置加`（轉題加效）`或`（轉題不加效）`。

### Step 6：整理同期声
- 将关键讲话汇总到`【同期聲】`部分（可选）。
- 翻译准确，保留原意。

### Step 7：写字幕
- Heading和Content各一行。
- 如有多个CG，用不同的ID_Number区分。

### Step 8：填写底部标记
- 以`櫔`开头。
- 按内容类型填写`((VT))`、`((LVO))`等标记。
- 结尾加`((END))`。

## 六、常见错误提醒（自检清单）

| 序号 | 错误类型 | 说明 |
|:---:|:---|:---|
| 1 | 口播结尾漏标记 | 忘记加`-VT-`或`-LVO-` |
| 2 | Clip标注缺描述 | 每个clip必须有画面描述括号 |
| 3 | 同期声翻译不准 | 外文同期声必须提供准确中文翻译 |
| 4 | 字幕颠倒 | Heading和Content内容信息重叠或位置颠倒 |
| 5 | 主观推断 | 自行添加"旨在应对XX"等未确认的归因 |
| 6 | 数字不规范 | 中英文数字混用，如"一萬"应写"1萬" |
| 7 | 译名不统一 | 外国人名未用规范译名 |
| 8 | 逻辑跳跃 | 叙事顺序混乱，缺乏过渡 |
| 9 | 信息冗余 | 正文与口播重复过多 |
| 10 | 脚标遗漏 | 底部`【全文文稿】`部分未正确填写 |
