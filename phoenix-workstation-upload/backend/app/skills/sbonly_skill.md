# SBONLY规范新闻稿撰写 Skill

## 基本信息

| 项目 | 内容 |
|------|------|
| **名称** | SBONLY新闻稿撰写（国际/涉台/国防/外交类） |
| **版本** | v2.0 |
| **适用范围** | 外交部、国台办、国防部例行记者会及重大新闻事件 |
| **输出格式** | 口播 + 正文(含clip) + 字幕 + SB + 全文文稿标记 |

## 一、触发条件

当用户提供以下素材时自动启用本Skill：
1. 外交部/国台办/国防部发言人答记者问全文
2. 重大国际事件新闻稿（含英文原声或外电素材）
3. 带有提问记者所属媒体信息（如凤凰卫视、法新社等）

## 二、输出格式规范

### 1. 【口播】

**撰写规则：**
- 首句格式：`【机构】發言人[姓名]，[日期]在例行記者會上，[針對事件/提問]作出回應。`
- **如提问记者来自凤凰卫视**：将"針對……作出回應"改为 **"回答鳳凰衛視記者提問時表示"**
- 其他媒体（法新社、路透等）：统一用"針對……作出回應"
- 提炼发言人核心表态，控制在150字以内
- 结尾用OUTCUE引用发言人原话关键词

**示例：**
```
國台辦發言人張晗，10日在例行新聞發佈會上回答鳳凰衛視記者提問時表示，賴清德揮霍台灣民眾血汗錢搞"軍援外交"，是巨資買偷渡，已成笑柄，充分暴露其"台獨"本性和偏執卑劣。
```

### 2. 【正文】

**格式：**
```
【正文】7-7164
SB [發言人姓名]
<Clip>[外电代号]_[编号]([时间码])</Clip>
([发言人原话全文])
<Clip>[外电代号]_[编号]([时间码])</Clip>
([发言人原话全文])
(cutaways:<Clip>[外电代号]_[编号]([时间码])</Clip> <Clip>[外电代号]_[编号]([时间码])</Clip>)
```

**规则：**
- Clip命名使用 **空格** 分隔（如 `APTN 1448_2`），而非下划线
- 时间码精确到秒（`05:50:06-06:05:00`）
- 正文中的同期声 **不加引号**，直接写内容
- 如发言人原话含英文，需 **提供中文翻译**，格式为：
  ```
  (英文) "English text..."
  (中文) 「中文翻譯...」
  ```
- cutaways至少提供2个备用画面时间码

### 3. 【字幕】——标题撰写规范

| 部分 | 格式 | 示例 |
|------|------|------|
| **Heading（上标）** | `主體 + 動作/事件` 客观陈述，不加立场评价词 | `長崎核爆資料館擬將"南京大屠殺"改為"南京事件"` |
| **Content（下标）** | `機構: 核心結論` 简洁有力，直接点明中方定性 | `中方:南京大屠殺鐵證如山 歷史不容翻案` |
| **Reporter** | 統一使用 `趙德民 綜合報道` | `趙德民 綜合報道` |

**标题长度控制：**
- Heading：建议12-20字
- Content：建议10-15字

**多种标题备选方案：**
当用户要求"多起几组主副标题"时，提供至少5组不同侧重角的标题组合，涵盖：
- 侧重事件本身
- 侧重中方定性/回应
- 侧重批判对方行为
- 侧重呼吁/敦促
- 侧重历史/法理依据

### 4. 【SB】——发言人信息

```
SB
<CG=左二行><T=>
Name01:[發言人姓名]
Title01:[機構]發言人
</CG>
```

**机构规范：**
- 外交部 → 中國外交部發言人
- 国台办 → 國台辦發言人
- 国防部 → 中國國防部新聞發言人
- 载人航天工程 → 中國載人航天工程新聞發言人

### 5. 【全文文稿】尾部标记

每篇稿件末尾必须包含以下标记：

```
【全文文稿】

櫔 (())
((VT))
((LVO))
((SB))
((UPSOUND))
((Standup))
((END))
```

## 三、特殊规则

### 1. 凤凰卫视记者提问处理

如素材明确显示提问记者来自 **凤凰卫视**：
- 【口播】必须使用"回答鳳凰衛視記者提問時表示"
- 正文中无需重复强调记者身份

### 2. 同期声英文处理

如发言人使用英文发言（或外电提供英文原声）：
- 保留英文原文（加引号）
- 另起一行提供中文翻译（用「」标注）
- 格式：`(英文) "..."` → `(中文) 「...」`

### 3. 国防部与外交部口径差异

- **外交部**：侧重国际法、国际关系、历史依据、和平呼吁
- **国防部**：侧重军事行动、主权安全、有力措施、严正交涉

## 四、新华社风格主副标题参考

如用户要求"新华社风格标题"或"多起几组标题"，按以下逻辑提供至少5组：

| 组别 | 侧重角度 | Heading | Content |
|------|----------|---------|---------|
| 1 | 事件定性 | 賴清德稱"頂住壓力突訪斯威士蘭" | 國台辦：巨資買偷渡成笑柄 暴露"台獨"本質 |
| 2 | 批判行为 | 賴清德大搞"軍援外交" 揮霍台灣民眾血汗錢 | 國台辦：不以為恥反以為榮 偏執卑劣本性畢露 |
| 3 | 揭露虚伪 | 民進黨當局高喊自由民主人權 | 國台辦：操控媒體、綠色恐怖、歧視陸配 人心盡失 |
| 4 | 法理反驳 | 賴清德妄稱台灣人"不願接受中國統治" | 國台辦：社會制度不同不是分裂藉口 認同一個中國均可商量 |
| 5 | 呼吁台胞 | 民進黨打著民主幌子謀"獨" | 國台辦：望台胞認清險惡用心 共同維護台海和平 |

## 五、操作流程

1. **接收素材** → 阅读全文，提取：发言人姓名、日期、机构、事件核心、记者提问（含媒体来源）
2. **确认记者身份** → 如提问者为凤凰卫视，口播加入"回答凤凰卫视记者提问时表示"
3. **撰写口播** → 提炼核心信息，附OUTCUE
4. **撰写正文** → 拆分发言人原话为2-3个Clip，标注时间码，添加cutaways
5. **撰写字幕标题** → 按Heading/Content格式，确保简洁有力
6. **填写SB信息** → 发言人姓名+机构
7. **添加尾部标记** → 完整保留【全文文稿】及所有标记

## 六、常见错误及规避

| 错误 | 正确做法 |
|------|----------|
| Clip用下划线 `APTN_1448_2` | 用空格 `APTN 1448_2` |
| 同期声加双引号 | 正文同期声不加引号 |
| Heading用"指"字开头 | 用客观陈述，不用"指" |
| Content用"中國外交部回應...:"冗长结构 | 用"中方:核心結論"简洁格式 |
| 凤凰卫视提问未在口播标注 | 口播必须体现"回答凤凰卫视记者提问" |
| 缺少cutaways | 至少提供2个备用画面 |
| 缺少尾部标记 | 必须完整保留【全文文稿】及标记 |
