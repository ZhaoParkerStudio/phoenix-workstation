# 干稿写作技能（Skill）说明

## 一、任务目标

将任意新闻素材（新华社、外电、政府公告、社交媒体等）压缩、提炼为 **"口播干稿"** 格式，适用于电视新闻口播、快讯播报场景。

## 二、输入要求

- 提供原始新闻材料（文字、链接、截图均可）。
- 可附带背景资料、数据、外电译稿等补充信息。

## 三、处理步骤

### 1. 通读全文，提取核心事实
- 5W1H：谁、何时、何地、何事、为何、如何。
- 优先保留：官方表态、关键数据、事发经过、最新进展。
- 剔除：主观评论、无实质信息的修饰语、重复内容。

### 2. 撰写干稿正文（口播内容）
- 开头直接陈述核心事实，不设铺垫。
- 语体：客观、平实、中性，使用书面语但适合朗读。
- 长度：通常 3~5 句话，控制在 200 字以内。
- 结尾以 `-END-` 标示。

### 3. 补充背景（如需）
- 若用户明确提出"补充背景"，则酌情添加 1~2 句与事件直接相关的背景信息（如历史背景、前期经过、数据对比）。
- 背景信息需源自素材本身，不额外臆造。

### 4. 生成【字幕】模板
- 标题（CG=干題）通常取事件核心人物+动作，或事件关键词。
- 使用标准 XML 或自定义标签格式：
  ```
  <CG=干題><T= >
  Content:（标题内容）
  </CG>
  ```

### 5. 调整与确认
- 若用户提出修改（如增加数据、调整表述、补充背景），按新要求重新生成，直至定稿。

## 四、输出格式（模板）

```
【口播】
（干稿）
（正文内容，简洁完整，末尾加 -END-）

【正文】
（一般为"无"或留空）

【字幕】
<CG=干題><T= >
Content:（标题）
</CG>
```

## 五、注意事项

- 不添加个人观点或猜测，仅陈述事实。
- 数字、时间、人名、地名必须与原始素材严格一致。
- 若有多源信息冲突，优先采用官方或权威来源，并在文中注明。
- 若用户未要求背景，默认不加，只写干稿。

## 六、示例

**输入素材**：朱鹤新会见巴西财政部副部长

**输出干稿**：

```
【口播】
（干稿）
中国人民银行副行长、国家外汇管理局局长朱鹤新10日会见来访的巴西财政部副部长阿伦卡斯特罗，双方就进一步发挥中巴可持续发展扩大产能合作基金作用、深化双边金融开放合作、加强可持续投资领域对话等议题进行了交流。国家外汇管理局副局长李红燕陪同会见。-END-

【正文】
无

【字幕】
<CG=干題><T= >
Content:朱鹤新会见巴西财政部副部长阿伦卡斯特罗
</CG>
```
